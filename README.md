# Сезоны

Внутренний сервис учёта сезонов программы НКО.

## Что это и для кого

Сервис заменяет Word-документы, в которых лидеры групп фиксировали итоги
сезонов. Вход — по общему паролю, без отдельных аккаунтов. Подробное
техническое задание см. в истории переписки с заказчиком.

## Текущий статус (этап 7 из 8)

Готово:
- Каркас проекта на Next.js, вход по общему паролю.
- Структура данных: филиалы, сезоны, столики, участники.
- Бюджет, вехи, выпускники, рекламная кампания, заметки сезона.
- Дашборд со сводной статистикой по всем сезонам.
- Визуальное оформление в стиле Alpha (см. техническое задание, раздел 7).
- PWA: установка на главный экран телефона, кеширование статики,
  автосохранение черновиков форм.
- Подготовка к деплою: страница проверки работоспособности `/api/health`,
  инструкция по деплою на хостинг — см. `DEPLOY.md`.

Дальше по плану: финальная проверка надёжности на мобильном/медленном
интернете.

## Как запустить локально (для разработки)

1. Установить зависимости: `npm install`
2. Заполнить `.env` (скопировать из `.env.example`, см. ниже) — нужны
   `DATABASE_URL` (строка подключения к PostgreSQL), `APP_PASSWORD` (пароль
   для входа в сервис) и `SESSION_SECRET` (любая длинная случайная строка).
3. Запустить сервер разработки: `npm run dev`
4. Открыть в браузере `http://localhost:3000` — откроется страница входа,
   нужно ввести пароль из `APP_PASSWORD`.

## Переменные окружения

См. файл `.env.example` — там описано, что нужно заполнить. Файл `.env` с
реальными значениями никогда не попадает в git (он в `.gitignore`).
