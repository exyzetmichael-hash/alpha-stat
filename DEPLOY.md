# Деплой на Timeweb Cloud — пошаговая инструкция

Эта инструкция для заказчика: как один раз настроить сервис на хостинге, и
что потом периодически проверять самому, без помощи программиста.

Сервис размещаем на **Timeweb Cloud** — российском хостинге с автоматическим
деплоем из Git и управляемой базой данных PostgreSQL. Все данные физически
хранятся в России.

---

## Часть 1. Один раз: регистрация и первый деплой

### Шаг 1. Регистрация на Timeweb Cloud

1. Откройте https://timeweb.cloud и нажмите «Регистрация».
2. Укажите email и телефон, подтвердите их кодом из SMS/письма.
3. Войдите в личный кабинет (https://timeweb.cloud/my).

На момент регистрации у Timeweb Cloud периодически бывают приветственные
бонусы для новых аккаунтов (пробные дни или баллы на баланс) — проверьте
раздел «Промокоды» / «Баланс» в личном кабинете, это поможет сэкономить на
старте. Условия меняются, поэтому уточняйте актуальную акцию на сайте при
регистрации.

### Шаг 2. Создание базы данных PostgreSQL

1. В личном кабинете откройте раздел **«Базы данных»** → «Создать базу данных».
2. Выберите **PostgreSQL** (последняя доступная версия, 15 или новее).
3. Выберите минимальный тариф — для 20-25 пользователей и нескольких
   десятков записей в год достаточно самого младшего платного тарифа
   (обычно это нагрузка в сотни рублей в месяц, как и заложено в бюджете).
4. Регион — любой российский (выбирается по умолчанию).
5. Дождитесь создания базы (обычно 1-2 минуты).
6. Откройте созданную базу и найдите **строку подключения** (Connection
   string / DSN) — она выглядит примерно так:
   ```
   postgresql://gen_user:ПАРОЛЬ@ХОСТ:5432/default_db
   ```
   Скопируйте и сохраните её в надёжном месте (например, менеджер паролей) —
   она понадобится на шаге 4.
7. Убедитесь, что в настройках базы включено **автоматическое резервное
   копирование** (на Timeweb Cloud у managed-баз это обычно включено по
   умолчанию одним переключателем «Бэкапы») — данные о бюджете и людях
   должны быть восстановимы при сбое.

### Шаг 3. Создание приложения (Cloud Apps)

1. В личном кабинете откройте раздел **«Облачные приложения»** (Cloud Apps)
   → «Создать приложение».
2. Выберите **«Из Git-репозитория»**.
3. Подключите свой GitHub-аккаунт (или другой провайдер Git, где лежит
   репозиторий сервиса) и выберите репозиторий проекта.
4. Укажите ветку для деплоя — обычно `main`.
5. Платформа должна автоматически распознать **Next.js** проект. Если
   попросит указать команды вручную, укажите:
   - Команда установки зависимостей: `npm install`
   - Команда сборки: `npm run build`
   - Команда запуска: `npm run start:prod`

   (`start:prod` — специальная команда проекта, которая перед запуском сама
   применяет нужные обновления структуры базы данных и не требует ручных
   действий при каждом обновлении сервиса.)
6. Выберите минимальный тариф, подходящий под небольшую нагрузку (несколько
   десятков пользователей) — обычно это самый младший платный план.

### Шаг 4. Переменные окружения

В настройках приложения найдите раздел **«Переменные окружения»**
(Environment variables) и добавьте три переменные:

| Имя | Значение |
|---|---|
| `DATABASE_URL` | строка подключения к базе из Шага 2, пункт 6 |
| `APP_PASSWORD` | пароль, который будут вводить все 20-25 лидеров для входа в сервис — придумайте свой |
| `SESSION_SECRET` | любая длинная случайная строка (40+ символов, можно сгенерировать на https://www.random.org/strings/ или в любом генераторе паролей) |

Сохраните переменные и запустите деплой (если он не запустился
автоматически после сохранения).

### Шаг 5. Первый деплой и проверка

1. Дождитесь окончания деплоя (Timeweb Cloud покажет лог сборки —
   дождитесь статуса «Запущено» / «Running»).
2. Платформа выдаст адрес вида `https://ваше-приложение.twc1.net` —
   откройте его в браузере.
3. Должна открыться страница входа сервиса. Введите пароль из `APP_PASSWORD`.
4. Проверьте, что сервис «видит» базу данных: откройте в браузере адрес
   `https://ваше-приложение.twc1.net/api/health` — должно появиться
   `{"status":"ok","database":"ok"}`. Если там ошибка — см. раздел
   «Если что-то не работает» ниже.

### Шаг 6 (опционально). Свой домен

Если у вас есть свой домен (например, `sezony-vashanko.ru`), в разделе
«Домены» приложения можно привязать его вместо технического адреса
`*.twc1.net`. Это не обязательно для работы сервиса.

## Часть 2. Что делать дальше — обновление сервиса

После первого деплоя всё дальнейшее обновление сервиса происходит
**автоматически**: как только в Git-репозиторий попадают новые изменения
в ветке `main`, Timeweb Cloud сам пересобирает и перезапускает приложение.
Вам не нужно ничего делать руками при обычных обновлениях кода.

---

## Часть 3. Что нужно периодически проверять самому

Раз в одну-две недели (или когда кто-то из лидеров пожалуется, что сервис
«не работает»):

1. **Откройте сервис в браузере.** Если страница входа открывается — всё в
   порядке.
2. **Откройте `/api/health`** (адрес сервиса + `/api/health`). Ответ
   `{"status":"ok","database":"ok"}` означает, что и сайт, и база данных
   работают.
3. **Проверьте раздел «Биллинг» / «Баланс»** в личном кабинете Timeweb
   Cloud — убедитесь, что на счету достаточно средств и автоматическое
   списание происходит без проблем. Это единственное, что может «внезапно»
   остановить сервис — заканчиваются деньги на балансе.

## Куда смотреть при проблеме

- **Сайт не открывается совсем (белый экран / ошибка соединения):**
  личный кабинет Timeweb Cloud → ваше приложение → раздел **«Логи»**
  (Logs) — там будут видны ошибки запуска. Также проверьте баланс счёта
  (см. выше).
- **Сайт открывается, но `/api/health` показывает ошибку
  (`"database": "unreachable"`):** проблема на стороне базы данных —
  личный кабинет → «Базы данных» → ваша база → убедитесь, что она имеет
  статус «Активна» (не приостановлена за неуплату или техническое
  обслуживание).
- **Не получается войти (пароль не принимается):** проверьте, что в
  переменных окружения приложения (`Часть 1, Шаг 4`) указан именно тот
  `APP_PASSWORD`, который вы сообщили лидерам — пароли регистра важны.
- **Если ничего не помогло:** соберите скриншот ошибки из раздела «Логи» и
  обратитесь к разработчику или в поддержку Timeweb Cloud (есть онлайн-чат
  в личном кабинете, обычно отвечают быстро).

## Бюджет (ориентир)

- Минимальный тариф Cloud Apps (приложение): ориентировочно около
  150-300 ₽/месяц.
- Минимальный тариф managed PostgreSQL: ориентировочно около 150-300 ₽/месяц.

Итого — в пределах нескольких сотен рублей в месяц, как и было заложено в
бюджете (раздел 6.4 технического задания). Точные актуальные цены смотрите
на странице тарифов https://timeweb.cloud при создании приложения и базы —
они могут немного измениться с момента написания этой инструкции.
