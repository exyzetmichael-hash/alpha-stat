# Деплой: Vercel + Neon (бесплатно)

Эта инструкция для заказчика: как один раз бесплатно разместить сервис и
что потом периодически проверять самому, без помощи программиста.

Сервис размещаем на двух бесплатных сервисах:

- **Vercel** — хостинг для самого сайта (приложения Next.js). Бесплатный
  тариф Hobby. Автоматически пересобирает сайт при изменениях в Git.
- **Neon** — бесплатная база данных PostgreSQL «в облаке».

> **Важно знать про бесплатный вариант (честно о минусах):**
> - Серверы Vercel и Neon находятся **не в России**. В техническом задании
>   было пожелание размещать всё в РФ — этот бесплатный вариант от него
>   отступает. Решение принято осознанно ради экономии; в сервисе хранятся
>   только имена участников, без «тяжёлых» персональных данных.
> - На бесплатном тарифе Neon база «засыпает» после ~5 минут простоя и
>   просыпается при первом обращении. Это значит, что **первая загрузка
>   после паузы может занять пару секунд** дольше обычного. Для внутреннего
>   рабочего инструмента это терпимо. Если в будущем захочется убрать эту
>   задержку — переходят на платный тариф или на российский хостинг
>   (отдельный разговор).

---

## Часть 1. Один раз: настройка

### Шаг 1. База данных Neon

1. Откройте https://neon.tech и зарегистрируйтесь (можно через тот же
   GitHub-аккаунт, где лежит код, — так проще).
2. Создайте проект (Create project): имя любое, регион выберите ближайший
   из доступных (например, Frankfurt / Europe — он географически ближе к
   России, чем США, загрузка будет чуть быстрее).
3. После создания Neon покажет **строку подключения** (Connection string),
   вида:
   ```
   postgresql://пользователь:пароль@ХОСТ.neon.tech/dbname?sslmode=require
   ```
   Скопируйте её и сохраните в надёжном месте (менеджер паролей) — она
   понадобится на шаге 3. Если Neon предлагает выбор «Pooled / Direct
   connection» — возьмите вариант с галочкой **«Direct connection»** (без
   слова `-pooler` в адресе): он нужен, чтобы при первом запуске сервис мог
   создать таблицы.
4. Бэкапы: на бесплатном тарифе Neon есть встроенная «история» базы
   (point-in-time restore) за несколько последних дней — отдельно ничего
   включать не нужно.

### Шаг 2. Создание приложения на Vercel

1. Откройте https://vercel.com и зарегистрируйтесь через GitHub-аккаунт,
   где лежит код сервиса.
2. Нажмите **«Add New…» → «Project»**.
3. В списке репозиториев найдите репозиторий сервиса и нажмите **«Import»**.
4. Vercel сам распознает, что это проект Next.js — менять команды сборки
   **не нужно** (проект уже настроен так, чтобы при сборке автоматически
   подготовить базу данных).
5. Пока **не нажимайте Deploy** — сначала добавьте переменные окружения
   (следующий шаг).

### Шаг 3. Переменные окружения

На той же странице импорта раскройте раздел **«Environment Variables»** и
добавьте три переменные:

| Имя (Key) | Значение (Value) |
|---|---|
| `DATABASE_URL` | строка подключения из Neon (Шаг 1, пункт 3) |
| `APP_PASSWORD` | пароль, который будут вводить все 20-25 лидеров для входа — придумайте свой |
| `SESSION_SECRET` | любая длинная случайная строка (40+ символов), например с https://www.random.org/strings/ |

После добавления всех трёх нажмите **«Deploy»**.

### Шаг 4. Первый деплой и проверка

1. Дождитесь окончания сборки (Vercel покажет лог; обычно 1-3 минуты,
   в конце — «Congratulations» / зелёная галочка).
2. Vercel выдаст адрес вида `https://имя-проекта.vercel.app` — откройте его
   в браузере.
3. Должна открыться страница входа. Введите пароль из `APP_PASSWORD`.
4. Проверьте, что сервис «видит» базу данных: откройте адрес
   `https://имя-проекта.vercel.app/api/health` — должно появиться
   `{"status":"ok","database":"ok"}`. Если там ошибка — см. раздел «Если
   что-то не работает» ниже.

### Шаг 5 (опционально). Начальные справочники

Сервис полностью работает и без этого шага: роли участников и категории
бюджета можно вводить вручную прямо в формах (вариант «Другое...»). Но если
хочется сразу заполнить стандартные списки (роли «Ведущий», «Помощник» и
т.д., категории бюджета), это делается один раз. Проще всего попросить об
этом разработчика — он запустит подготовленный скрипт начальных данных,
указав на вашу базу Neon. Отдельных действий с вашей стороны не требуется.

### Шаг 6 (опционально). Свой домен

Если есть свой домен — в настройках проекта Vercel, раздел **«Domains»**,
можно привязать его вместо адреса `*.vercel.app`. Для работы сервиса это не
обязательно.

## Часть 2. Обновление сервиса

После первой настройки всё обновляется **автоматически**: как только в
Git-репозиторий (ветка `main`) попадают новые изменения, Vercel сам
пересобирает и публикует сайт. Вручную делать ничего не нужно.

---

## Часть 3. Что проверять самому периодически

Раз в одну-две недели (или если кто-то из лидеров пожаловался, что сервис
«не работает»):

1. **Откройте сервис в браузере.** Открылась страница входа — всё в порядке.
   (Помните: если базой давно не пользовались, первая загрузка может быть на
   пару секунд медленнее — это нормально для бесплатного тарифа.)
2. **Откройте `/api/health`** (адрес сервиса + `/api/health`). Ответ
   `{"status":"ok","database":"ok"}` означает, что и сайт, и база работают.

На бесплатных тарифах денег ни с Vercel, ни с Neon не списывается, поэтому
следить за балансом не нужно.

## Если что-то не работает

- **Сайт не открывается совсем:** зайдите в панель Vercel
  (https://vercel.com) → ваш проект → вкладка **«Deployments»** → откройте
  последний деплой → **«Logs»**. Там видно, на каком шаге ошибка.
- **Сайт открывается, но `/api/health` показывает
  `"database": "unreachable"`:** проблема со стороны базы — зайдите в
  https://neon.tech, откройте проект и убедитесь, что он активен (не удалён,
  не приостановлен). Также проверьте, что переменная `DATABASE_URL` в
  настройках Vercel (Project → Settings → Environment Variables) совпадает
  со строкой подключения из Neon.
- **Не получается войти (пароль не принимается):** проверьте, что в
  переменных окружения Vercel указан именно тот `APP_PASSWORD`, который вы
  сообщили лидерам (регистр букв важен). После изменения переменной нужно
  заново задеплоить проект (Vercel → Deployments → «Redeploy»).
- **Если ничего не помогло:** сделайте скриншот ошибки из «Logs» и
  обратитесь к разработчику.

## Стоимость

- Vercel Hobby: **бесплатно** (для внутреннего инструмента на 20-25 человек
  лимиты бесплатного тарифа с большим запасом достаточны).
- Neon Free: **бесплатно**.

Итого — **0 ₽/месяц**. Плата за тарифы Vercel/Neon может понадобиться только
при кратном росте числа пользователей или объёма данных, что для этого
сервиса не ожидается.
